# AI音乐提示词生成器 v4.2（整合版）

## 概述

本Skill整合了两个音乐生成器的核心优势：
- **ai-music-prompt-generator** 的对话收集和模型选择功能
- **ai-music-prompts** 的18大风格方案和浏览器自动化功能

**核心亮点**：
- 🎤 **智能对话收集**：通过问答精准捕捉创作需求
- 🎵 **18大风格方案**：覆盖各种音乐风格的完整配置
- 🤖 **浏览器自动化**：自动操作妙响平台完成歌曲生成
- 🎯 **多模型支持**：支持妙响平台8个AI模型选择

**兼容环境**：
- **SOLO 独立端**：优先使用内置 `browser_*` 工具自动化操作
- **TRAE IDE**：自动检测后打开用户电脑自带浏览器

## 第一步：收集创作需求

按顺序询问用户以下问题。**如果用户已在初始消息中回答了某个问题，跳过该问题，只问缺失的：**

### 1.1 基础创作信息

1. **创作主题**：你想创作一首什么主题的歌曲？（例：毕业季、失恋、深夜加班回家、古风、城市旅行、夏日周末等）
2. **情绪氛围**：希望歌曲传达什么样的情绪？（例：温暖治愈、热血燃、伤感催泪、松弛慵懒、青春活力、安静治愈等）
3. **音乐风格**：有特定的音乐风格偏好吗？（例：抒情流行、民谣、摇滚、R&B、国风、City Pop、电子、爵士等）
4. **是否纯音乐**：想要带有人声的歌曲，还是纯音乐（无人声，只有乐器演奏）？

### 1.2 如果选择"带有人声"，继续询问：

5. **人声要求**：希望是男声还是女声？有什么音色要求？（例：清澈女声、磁性男声、空灵、慵懒、有力量感等）
6. **歌词方向**：有没有想要的歌词主题或记忆点？（例：某句核心句子、想写的歌词主题，或"请AI自由创作"）

### 1.3 平台配置

7. **创作模式**：你想用哪种模式创作？
   - **灵感创作**：简单输入描述，AI自动优化提示词后生成（简单快速，新手推荐）
   - **专业模式**：可以选择风格标签、调整风格权重、调整创意发散程度（精准控制，有经验用户推荐）

8. **AI模型选择**：想用哪个AI模型来生成？（根据平台8个模型的特点选择）

   | 模型 | 特点 | 适用场景 |
   |------|------|---------|
   | **Sodance v1.5** ⭐ | 专注音乐性表达，编曲层次丰富，华语流行创意 | 通用推荐，尤其是抒情、流行 |
   | **Sway v5.5** | 音乐性和音质俱佳，情绪表达细腻，提示词遵循能力强 | 情感细腻、需要精准控制 |
   | **Mureka v9** | 华丽编曲技巧，贴合欧美流行音乐制作 | 欧美流行、复杂编曲 |
   | **MiniMax v2.6** | 人声情绪丰富，演唱功力细致 | 人声表现要求高的歌曲 |
   | **TemPolor v4.1a** | 音乐性和音质俱佳，情绪表达细腻 | 通用，支持快速试听 |
   | **音潮 v3.0** | 支持音频续写和仿写，擅长欧美/华语流行 | 需要续写或仿写的场景 |
   | **SeedMusic v4.3+** | 整体效果稳定，人声音质好，国风优异 | 国风、古风、稳定输出 |
   | **TemPolor v4.0** | 语种丰富，歌声旋律灵动，音质纯净 | 多语种、旋律要求高 |

#### 方案A：SOLO内置浏览器模式（完整自动化）

**A1. 打开妙响平台**
```
browser_navigate → https://music.douyin.com/studio
等待页面加载完成（最多10秒）
```

**A2. 登录状态检测**
```
获取页面快照，检查关键元素：
- 如果看到"登录"按钮或扫码提示 → 未登录
- 如果看到创作输入框或用户头像 → 已登录
- 如果页面加载失败 → 网络异常

[未登录处理]：
1. 截图当前页面，显示登录二维码
2. 提示用户："请在浏览器中扫码登录抖音账号，完成后点击'已登录'"
3. 使用 AskUserQuestion 等待用户确认
4. 用户确认后，刷新页面重新检测

[网络异常处理]：
1. 重试3次，每次间隔5秒
2. 如果仍失败 → 回退到手动复制模式
```

**A3. 创作模式选择检测**
```
检查当前页面状态：
- 如果在首页（有"灵感创作"和"专业模式"按钮）：
  → 点击用户选择的模式按钮
- 如果已在创作页面（有输入框和生成按钮）：
  → 直接进入A4步骤
- 如果页面结构异常：
  → 截图记录，回退到手动模式

```

```
定位创作灵感输入框：

填入后验证：
- 获取输入框 value，确认提示词已正确填入

```

## 检测要点详解

### 登录状态检测逻辑
```
获取页面快照后，检查以下元素：
1. 查找"登录"、"扫码"、"请登录"等关键词
2. 查找用户头像、用户名等已登录标识
3. 查找创作输入框（通常只有登录后才显示）

判断规则：
- 同时存在登录提示且不存在创作功能 → 未登录
- 存在创作输入框或用户头像 → 已登录
- 页面结构异常 → 需要刷新重试
```

### 元素定位策略
```
优先级1：通过 placeholder 文本定位
- "输入你的创作灵感"
- "输入你的灵感"
- "例如：高燃踩点硬曲"

优先级2：通过元素类型和位置
- textbox 类型，位于页面中央偏上
- 带有特定 class 或 id（如"input-box"、"prompt-input"）

优先级3：通过相邻元素定位
- 找到"生成"按钮，其上方/左侧的输入框

备用方案：如果以上都失败
- 截图当前页面
- 使用 browser_evaluate 执行 JavaScript 查找
- 提示用户手动操作
```

### 网络异常处理
```
重试策略：
1. 第一次失败：等待5秒，重试
2. 第二次失败：等待10秒，重试
3. 第三次失败：回退到手动模式

常见网络问题：
- 页面加载超时 → 检查网络连接
- 元素未找到 → 页面可能未完全加载
- 点击无响应 → 元素可能被遮挡或未启用
```

## 注意事项

1. **标签冲突避免**：不要同时选择矛盾标签（如"高燃+松弛"）
2. **权重平衡**：风格权重过高可能导致音乐过于风格化而缺乏旋律性
3. **描述具体化**：越具体的场景描述，AI生成的画面感越强
4. **记忆点设计**：好的记忆点通常包含具体意象+情感共鸣
5. **BPM适配**：根据使用场景选择合适的节奏速度
6. **中文描述**：妙响偏好中文自然语言描述
7. **浏览器环境**：自动检测 SOLO/TRAE 环境，选择最佳操作方式
8. **登录状态**：自动化操作前确保已登录抖音账号
9. **纯音乐模式**：必须在生成前确认勾选了"纯音乐"选项
10. **模型选择**：不同模型擅长不同风格，参考推荐配置
11. **异常处理**：任何步骤失败都会自动重试或回退到手动模式
12. **截图记录**：关键步骤失败时会截图，便于排查问题
