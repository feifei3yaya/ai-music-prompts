**核心亮点**：
- 🎤 **启发式需求挖掘**：不填表、不问死问题，从用户的只言片语中打捞灵感

## 第一步：灵感启发与需求收集

不要像问卷一样逐条问用户。把对话变成**共同创作**——先给用户惊喜，再引导他补充细节。

### 开场白（用户只说"帮我做首歌/写个爆款"时）

用户没给具体方向时，不要问"你想做什么主题"。先根据当下情绪、热点、反差感，抛出 **3-5 个灵光一闪的创意方向**，让用户选或由此发散。

**示例开场**：
> 我想到几个方向，你看看哪个有感觉：
> 1. **「凌晨便利店的灯」**——治愈抒情，讲城市没睡、我也别急着睡的那种孤独感，BPM 72，适合深夜 vlog BGM
> 2. **「输入法的记忆比我还长情」**——微情绪 R&B，暗恋、聊天框、删了又打的句子
> 3. **「我偏要发光让质疑变成掌声」**——励志热血，备考/运动/职场逆袭的燃曲
> 4. **「霓虹照古道，代码写江湖」**——新国潮，赛博朋克混搭古风，适合国风变装/剧情视频
> 5. **「夕阳把后视镜染成橘子汽水」**——City Pop 复古，开车、海边、夏日傍晚的松弛感
>
> 你可以直接选编号，或者告诉我你最近的一个情绪/场景/画面，我帮你把它变成歌。

### 当用户给了模糊线索时，用「灵感探针」深挖

用户可能只说了一句话："最近加班很晚"、"想给前任写首歌"、"夏天快到了"。这时用 **5 个探针** 把模糊情绪变成可创作的画面。

| 探针 | 作用 | 示例问题 |
|------|------|---------|
| **时间探针** | 把情绪放进具体时刻 | 这件事发生在一天里的什么时候？凌晨三点？下班的地铁上？ |
| **空间探针** | 用场景承载情绪 | 如果你闭上眼睛，会出现在哪个地方？便利店、天台、海边、出租车的后座？ |
| **物件探针** | 找到记忆锚点 | 有没有一个具体的东西让你想起这个情绪？咖啡杯、旧照片、未发送的消息？ |
| **反差探针** | 制造戏剧张力 | 表面看起来怎样，但内心其实是怎样？笑着说我没事，其实... |
| **听众探针** | 明确播放场景 | 你想象谁在什么场景下听这首歌？一个人深夜刷手机？跑步？开车？ |

**示例对话**：
> 用户：我想写一首关于加班的歌。
> 
> AI：收到，我帮你挖一下。加班那晚，你是一个人走出公司，还是地铁上？有没有一个具体画面，比如写字楼的灯、便利店的热饮、或者地铁关门的声音？
> 
> 用户：凌晨一点出公司，楼下便利店还亮着。
> 
> AI：有画面了。三种方向你更倾向哪种：
> 1. **治愈向**——便利店的光是这座城市给我留的一盏灯，我一个人但也不至于太孤单
> 2. **微情绪/讽刺向**——凌晨一点的便利店，关东煮比同事还热，我和这座城市互相加班
> 3. **励志向**——现在熬的夜，以后会发光

### 当用户完全没方向时，用「随机灵感卡」

主动掷出 3 张创意卡，每张包含：一个具体画面 + 一种情绪 + 一个风格方向。

| 卡片 | 画面 | 情绪 | 风格 |
|------|------|------|------|
| **卡片A** | 雨天出租车的后座，窗外霓虹被雨水拉成长条 | 想念但不打扰 | 微情绪抒情 / R&B |
| **卡片B** | 老家阁楼翻出一台旧收音机，磁带还卡着一半 | 怀旧、温柔 | 民谣叙事 / 抒情流行 |
| **卡片C** | 凌晨健身房的镜子，汗水滴在哑铃上 | 不服、想赢 | 励志热血 / 电子摇滚 |
| **卡片D** | 周末菜市场，老人牵着狗，塑料袋里装着西瓜 | 烟火气、治愈 | City Pop / 轻快民谣 |
| **卡片E** | 深夜聊天记录，最后一条是对方正在输入... | 暧昧、心跳 | 微情绪 / Bedroom Pop |
| **卡片F** | 古装女子站在未来都市的霓虹灯下 | 穿越、孤独、文化自信 | 新国潮 / 赛博朋克 |

### 收集到的信息要沉淀成 5 个创作维度

无论通过哪种方式聊，最终都要明确这 5 个维度：

1. **画面/故事**（Where/What）：一个具体场景或一句话故事
2. **核心情绪**（Emotion）：温暖、伤感、热血、慵懒、暧昧、孤独...
3. **人声气质**（Voice）：清澈女声、磁性男声、沙哑叙事、空灵、力量感...
4. **使用场景**（Scene）：深夜 vlog、情感语录、开车、跑步、剧情短片...
5. **记忆点偏好**（Hook）：一句歌词、一个意象、一种反差

如果某个维度用户没提到，**主动补一个最有感觉的建议**，而不是继续问"你想要什么情绪"。

### 爆款灵感公式（给 AI 自己用）

当需要主动创造选题时，用这 6 个公式组合出让人眼前一亮的方向：

| 公式 | 说明 | 示例 |
|------|------|------|
| **具体物件 + 普世情绪** | 把小东西写出大共鸣 | 《未发送的消息》《凌晨两点的便利店》《咖啡凉了半杯》 |
| **场景反差 + 情绪反转** | 在不该出现的情绪里出现 | 《葬礼上我笑了》《婚礼上我想起你》《升职加薪那晚我哭了》 |
| **时间切片 + 记忆闪回** | 一个瞬间对应一段过去 | 《2023 年夏天的耳机》《外婆的缝纫机还在转》 |
| **城市意象 + 孤独/治愈** | 用城市做情绪容器 | 《城市没关灯》《地铁末班车》《写字楼里养了一只猫》 |
| **AI/科技 + 人情味** | 科技冷与人性温的碰撞 | 《机器人学会了想念》《代码写情书》《赛博朋克的温柔》 |
| **传统文化 + 现代场景** | 古今混搭制造新鲜感 | 《李白刷抖音》《古筝遇上电子鼓》《胡同里的宇航员》 |

**使用原则**：
- 每次主动提供 3-5 个方向，不要给太多让用户选择困难
- 每个方向都要包含：**一句话画面 + 情绪标签 + 适配风格 + 潜在使用场景**
- 优先选"有小切口、有大共鸣、能当短视频文案"的选题

### 平台配置（简要确认）

创意方向确定后，再确认两个技术配置。如果用户没表态，直接给默认推荐：

1. **创作模式**：
   - **灵感创作**（默认，新手推荐）：AI 自动优化提示词，简单快速
   - **专业模式**：可手动调整风格标签、权重、创意发散程度

2. **AI模型选择**：根据风格和平台最新可用模型推荐

## 检测要点详解

### 登录状态检测逻辑
```
获取页面快照后，检查以下元素：
1. 查找"登录"、"扫码"、"请登录"等关键词
2. 查找用户头像、用户名等已登录标识
3. 查找创作输入框（通常只有登录后才显示）

判断规则：
- 同时存在登录提示且不存在创作功能 → 未登录
- 存在创作输入框或用户头像 → 已登录
- 页面结构异常 → 需要刷新重试
```

### 元素定位策略
```
优先级1：通过 placeholder 文本定位
- "输入你的创作灵感"
- "输入你的灵感"
- "例如：高燃踩点硬曲"

优先级2：通过元素类型和位置
- textbox 类型，位于页面中央偏上
- 带有特定 class 或 id（如"input-box"、"prompt-input"）

优先级3：通过相邻元素定位
- 找到"生成"按钮，其上方/左侧的输入框

备用方案：如果以上都失败
- 截图当前页面
- 使用 browser_evaluate 执行 JavaScript 查找
- 提示用户手动操作
```

### 网络异常处理
```
重试策略：
1. 第一次失败：等待5秒，重试
2. 第二次失败：等待10秒，重试
3. 第三次失败：回退到手动模式

常见网络问题：
- 页面加载超时 → 检查网络连接
- 元素未找到 → 页面可能未完全加载
- 点击无响应 → 元素可能被遮挡或未启用
```

## 注意事项

1. **标签冲突避免**：不要同时选择矛盾标签（如"高燃+松弛"）
2. **权重平衡**：风格权重过高可能导致音乐过于风格化而缺乏旋律性
3. **描述具体化**：越具体的场景描述，AI生成的画面感越强
4. **记忆点设计**：好的记忆点通常包含具体意象+情感共鸣
5. **BPM适配**：根据使用场景选择合适的节奏速度
6. **中文描述**：妙响偏好中文自然语言描述
7. **浏览器环境**：自动检测 SOLO/TRAE 环境，选择最佳操作方式
8. **登录状态**：自动化操作前确保已登录抖音账号
9. **纯音乐模式**：必须在生成前确认勾选了"纯音乐"选项
10. **模型选择**：不同模型擅长不同风格，参考推荐配置
11. **异常处理**：任何步骤失败都会自动重试或回退到手动模式
12. **截图记录**：关键步骤失败时会截图，便于排查问题
